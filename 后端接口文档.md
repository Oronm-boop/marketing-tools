# Backend 接口文档（端口与连接详解）

本文档基于代码仓库当前实现自动整理，覆盖：
- FastAPI **REST 接口**（前端调用）
- MCP **Streamable HTTP** 接口（Agent/工具调用）
- 端口、依赖服务、连接方式、环境变量

> 默认无鉴权（未实现 Token/Session），请仅在可信网络使用。

---

## 1. 服务与端口总览（“端口与连接”）

### 1.1 REST API（FastAPI）

- **服务进程**：`backend/main.py`
- **监听地址**：`0.0.0.0`
- **监听端口**：`20090`
- **Base URL（本机）**：`http://127.0.0.1:20090`
- **OpenAPI/Swagger**
  - Swagger UI：`/docs`
  - ReDoc：`/redoc`

### 1.2 MCP Server（Streamable HTTP）

- **服务进程**：`backend/mcp_server.py`
- **监听地址**：默认 `0.0.0.0`
- **监听端口**：默认 `8000`（可用 `--port` 改）
- **MCP 入口路径**：`/mcp`
- **Base URL（本机默认）**：`http://127.0.0.1:8000/mcp`
- **启动脚本（Windows）**：`backend/start_mcp_server.bat`

### 1.3 依赖服务端口（默认/示例）

这些端口来自 `backend/.env` 与 `backend/docker-compose.yaml`（以及仓库根 `docker-compose.yaml`）。

#### 向量数据库：Qdrant

- **Qdrant HTTP**：`localhost:16333`
  - Docker 映射：`16333:6333`
- **Qdrant gRPC**：`localhost:16334`
  - Docker 映射：`16334:6334`
- **健康检查**：`http://localhost:16333/health`

#### 图数据库：Neo4j（可选）

- **Neo4j HTTP**：`localhost:7474`（Docker 映射 `7474:7474`）
- **Neo4j Bolt**：`bolt://localhost:17687`（Docker 映射 `17687:7687`）
- **默认账号**：`neo4j / password`（见 `backend/.env` 与 `backend/docker-compose.yaml`）

#### 图数据库：NebulaGraph（可选）

- **NebulaGraph GraphD**：默认 `127.0.0.1:9669`
- **账号/空间**：见 `backend/.env`（`NEBULAGRAPH_*`）

#### Embedding：本地 OpenAI 兼容服务（必须）

用于 `/embed`、`/query`、以及一键流程中的向量化。

- **Base URL**：`LOCAL_OPENAI_BASE_URL`（示例：`http://192.168.1.102:1234/v1`）
- **模型**：`LOCAL_OPENAI_MODEL`（示例：`text-embedding-bge-m3`）
- **维度**：`LOCAL_OPENAI_DIMENSION`（示例：`1024`）

#### OCR/VL（可选）

解析器可能会用到（取决于文件类型与解析策略）。

- **Base URL**：`LOCAL_VL_BASE_URL`（示例：`http://127.0.0.1:15234/v1`）
- **模型**：`LOCAL_VL_MODEL`

#### MinerU（可选，容器）

仓库根 `docker-compose.yaml` 中包含 `mineru` 示例服务：
- `7860:7860`
- `9001:8000`

## 2. 环境变量（连接配置）

文件：`backend/.env`（示例）

### 2.1 Embedding（必须）

- `LOCAL_OPENAI_BASE_URL`
- `LOCAL_OPENAI_API_KEY`（本地通常可随便填）
- `LOCAL_OPENAI_MODEL`
- `LOCAL_OPENAI_DIMENSION`

### 2.2 Qdrant（必须）

- `QDRANT_HOST`（默认 `localhost`）
- `QDRANT_PORT`（默认 `16333`）
- `QDRANT_URL`（可选：云端 URL）
- `QDRANT_API_KEY`（可选）

### 2.3 Neo4j / NebulaGraph（可选）

- Neo4j
  - `NEO4J_URI`（默认 `bolt://localhost:17687`）
  - `NEO4J_USER`
  - `NEO4J_PASSWORD`
- NebulaGraph
  - `NEBULAGRAPH_HOSTS`（默认 `127.0.0.1:9669`，逗号分隔）
  - `NEBULAGRAPH_USER`
  - `NEBULAGRAPH_PASSWORD`
  - `NEBULAGRAPH_SPACE`

### 2.4 文件上传限制（REST API 侧）

可通过环境变量覆盖（文件：`backend/utils/config.py`）：
- `MAX_FILE_SIZE_MB`：默认 `5120`（约 5GB）
- `MAX_FILES_PER_UPLOAD`：默认 `10`

也可以调用接口查看：`GET /config/limits`

## 3. 启动方式与连通性检查

### 3.1 启动依赖服务（推荐：Qdrant + Neo4j）

在 `backend/` 目录下：

```bash
docker-compose up -d
```

端口映射见 `backend/docker-compose.yaml`：
- Qdrant：`16333`（HTTP）、`16334`（gRPC）
- Neo4j：`7474`（HTTP）、`17687`（Bolt）

### 3.2 启动 REST API（FastAPI）

在 `backend/` 目录下（确保已安装依赖并配置 `.env`）：

```bash
python main.py
```

启动后：
- `GET http://127.0.0.1:20090/health`
- `Swagger: http://127.0.0.1:20090/docs`

### 3.3 启动 MCP Server（可选）

在 `backend/` 目录下：

```bash
python mcp_server.py --host 0.0.0.0 --port 8000
```

或直接运行：`backend/start_mcp_server.bat`

## 4. REST API（FastAPI）接口清单

### 4.1 约定

- **Content-Type**
  - JSON：`application/json`
  - 文件上传：`multipart/form-data`
- **CORS**：允许所有来源（`*`）
- **错误格式**：大多为 FastAPI 默认 `{"detail": ...}`；部分接口直接返回自定义 dict

### 4.2 健康与配置

#### GET `/health`

检查后端服务依赖（Qdrant、Neo4j）可用性。

- **响应示例**

```json
{
  "status": "healthy",
  "services": {
    "qdrant": { "status": "healthy" },
    "neo4j": { "status": "healthy" }
  },
  "config": {
    "max_file_size": "5.00 GB",
    "max_files_per_upload": 10
  }
}
```

#### GET `/config/limits`

获取上传限制与支持格式。

---

### 4.3 文件解析（Parse）

#### POST `/parse`

上传并解析文件，结果落盘到 `data/parsed/{file_id}.json`，同时返回元素预览（最多前 10 个）。

- **请求**：`multipart/form-data`
  - `file`：文件（必填）
  - `collection_name`：集合名（默认 `default_collection`）
  - `file_id`：可选（强烈建议前端传入以保证全链路一致）
  - 清洗选项（均为 bool，默认 True）：
    - `use_advanced_clean`
    - `merge_broken_sentences`
    - `remove_headers_footers`
    - `normalize_bullets`
    - `fix_ocr_errors`

- **curl 示例**

```bash
curl -X POST "http://127.0.0.1:20090/parse" ^
  -F "file=@D:\path\doc.pdf" ^
  -F "collection_name=default_collection" ^
  -F "use_advanced_clean=true"
```

- **响应字段（关键）**
  - `file_id`、`filename`、`file_type`、`parser_type`
  - `element_count`、`elements`（预览）、`result_file`

---

### 4.4 分块（Chunk）

#### POST `/chunk`

对 parse 的 `elements` 做分块，落盘到 `data/chunks/chunks_{file_id}.json`。

- **请求 JSON**
  - `elements`: array（必填）
  - `file_id`: string（可选，不传会生成）
  - `collection_name`: string（默认 `default_collection`）
  - `method`: `"semantic"` 或 `"element_grouping"`（默认 `"semantic"`）
  - `chunk_size`: int（默认 1000）
  - `chunk_overlap`: int（默认 200）

---

### 4.5 向量化（Embed）

#### POST `/embed`

将 `chunks` 向量化并写入 Qdrant。

- **请求 JSON**
  - `chunks`: array（必填）
  - `collection_name`: string（默认 `default_collection`）
  - `provider`: 参数存在但服务端强制使用 `local_openai`

- **响应（关键）**
  - `collection_name`
  - `vector_count`
  - `vector_dimension`

> 注意：如果 Qdrant 集合已存在但维度与当前 embedding 模型维度不一致，会返回 `400`，提示换集合名或删除旧集合。

---

### 4.6 知识图谱（KG）

#### POST `/kg/extract`

从文本抽取实体/关系/三元组，落盘到 `data/kg/kg_{file_id}.json`（若未传 `file_id` 会生成）。

- **请求 JSON**
  - `text`: string（必填）
  - `ner_method`: `"spacy" | "hanlp" | "paddle"`（默认 `"spacy"`）
  - `re_method`: `"rule_based" | "model_based"`（默认 `"rule_based"`）
  - `file_id`: string（可选）
  - `collection_name`: string（默认 `default_collection`）

#### POST `/kg/store`

把三元组写入图数据库（Neo4j 或 NebulaGraph）。

- **请求 JSON**
  - `triples`: array（必填）
  - `provider`: `"neo4j" | "nebula"`（默认 `"neo4j"`）
  - `collection_name`: string（默认 `"default"`，与其他接口默认值不同）

#### POST `/kg/embed`

对图节点向量化（node2vec/graphsage）。

- **请求 JSON**
  - `nodes`: array
  - `edges`: array
  - `method`: `"node2vec" | "graphsage"`（默认 `"node2vec"`）
  - `dimensions`: int（默认 128）

---

### 4.7 RAG 查询

#### POST `/query`

向量检索 +（可选）重排序 +（可选）KG 子图上下文。

- **请求 JSON**
  - `query`: string（必填）
  - `collection_name`: string（默认 `default_collection`）
  - `top_k`: int（默认 10）
  - `rerank_top_k`: int（默认 10；设为 0 可禁用）
  - `use_kg`: bool（默认 False）
  - `use_reranking`: bool（默认 True）
  - `graph_db_provider`: string（可选，如 `"neo4j"`）

- **响应（关键）**
  - `vector_results`、`reranked_results`（包含 `text/score/metadata`）
  - `kg_context`（可选）
  - `context`（合并后的上下文字符串）

---

### 4.8 集合（Collection）与文件管理

#### GET `/collections`

列出所有 Qdrant 集合（包含 points_count、vectors_count 等）。

#### GET `/collections/{collection_name}/info`

查询集合详情与统计（可选包含图数据库统计）。

- Query 参数：
  - `include_kg`: bool（默认 False）
  - `graph_db_provider`: string（默认 `"neo4j"`）

#### POST `/collections/create`

创建集合。

- **请求 JSON**
  - `collection_name`: string（必填）
  - `vector_size`: int（默认 1024）
  - `distance`: string（默认 `"Cosine"`）
  - `recreate_if_exists`: bool（默认 False）

#### DELETE `/collections/{collection_name}`

完整删除集合数据（向量库 + 可选图数据库 + 可选本地文件）。

- Query 参数：
  - `delete_kg`: bool（默认 True）
  - `delete_files`: bool（默认 True）
  - `graph_db_provider`: string（默认 `"neo4j"`）

#### GET `/kg/collections`

列出图数据库中的集合（按 `collection` 字段隔离）。

- Query 参数：`provider`（默认 `"neo4j"`）

#### DELETE `/kg/collections/{collection_name}`

删除图数据库中某集合的数据。

- Query 参数：`provider`（默认 `"neo4j"`）

#### GET `/parsed/files`

列出已解析文件（支持按 `collection_name` 过滤）。

#### GET `/parsed/files/{filename}`

读取指定 parsed 文件内容（带路径遍历安全校验）。

#### GET `/chunks/files`

列出 chunk 文件（支持按 `collection_name` 过滤）。

#### GET `/chunks/files/{filename}`

读取指定 chunk 文件内容（带路径遍历安全校验）。

#### GET `/kg/files`

列出 KG 文件（支持按 `collection_name` 过滤）。

#### GET `/kg/files/{filename}`

读取指定 KG 文件内容（带路径遍历安全校验）。

#### DELETE `/files/{file_id}`

按 `file_id` 删除该文件在：
- Qdrant 中的向量（按 `metadata.file_id` filter）
- 图数据库中的节点/关系（可选）
- 本地 `data/parsed|chunks|kg` 文件（可选）

Query 参数：
- `collection_name`（默认 `default_collection`）
- `delete_from_kg`（默认 True）
- `delete_local_files`（默认 True）
- `graph_db_provider`（默认 `"neo4j"`）

#### POST `/files/batch-delete`

批量删除多个 `file_id` 的数据。

- **请求 JSON**
  - `file_ids`: string[]
  - `collection_name`: string（默认 `default_collection`）
  - `delete_from_kg`: bool（默认 True）
  - `delete_local_files`: bool（默认 True）
  - `graph_db_provider`: string（默认 `"neo4j"`）

---

### 4.9 一键处理（上传->解析->分块->向量->KG）

#### POST `/process-all`

上传文件后执行全流程（落盘 parsed/chunks/kg，并写入 Qdrant；KG 存储失败不会让整体失败）。

- **请求**：`multipart/form-data`
  - `file`：文件（必填）
  - `collection_name`：默认 `default_collection`
  - `file_id`：可选
  - 分块参数：`chunk_method` / `chunk_size` / `chunk_overlap`
  - KG 参数：`ner_method` / `re_method` / `graph_db_provider`
  - 清洗参数：同 `/parse`

## 6. 常见连通性问题（按“连接/端口”排查）

### 6.1 REST API 正常，但 /health 显示 qdrant unhealthy

- 检查 Qdrant 是否在 `QDRANT_HOST:QDRANT_PORT` 可访问（默认 `localhost:16333`）
- 如果用 docker-compose（`backend/docker-compose.yaml`），确认端口映射：
  - 容器内 6333 → 宿主 16333

### 6.2 /embed 或 /query 报 embedding 相关错误

- 检查 `.env` 的 `LOCAL_OPENAI_BASE_URL` 是否可访问（示例 `http://192.168.1.102:1234/v1`）
- 确认本地 OpenAI 兼容服务是否提供 `POST /embeddings`
- 如果集合维度不匹配：换一个新的 `collection_name`，或删除旧集合再创建

### 6.3 Neo4j 连接失败

- 默认 Bolt：`bolt://localhost:17687`
- docker-compose 映射：宿主 `17687` → 容器 `7687`
- 默认账号：`neo4j / password`
